一、考前准备
1. AI 编程工具：已安装Cursor等Native IDE，准备好 Claude Code / Gemini-Pro 等高级账号模型（本项目要求自带模型账号）。
2. Docker 环境：本地已安装 Docker Desktop 并处于运行状态（核心考核点）。
3. GitHub 账号：用于创建仓库和自动化测试。

二、题目菜单
题目名称：看板任务管理
核心技术栈：React/Vue/原生 + Python/Java/Go + DB
交付红线：全栈一键启动（Docker）

三、详细业务需求
全栈：看板任务管理（KanBan Board）
· 目标：实现类似 Trello 的任务拖拽（Todo/Doing/Done）及数据同步。
· 技术：Web 前端 + 后端 API + 数据库。
· 交付：前后端及数据库必须通过 `docker compose` 一键联调。

四、交付标准与红线（The Gloden Rules）
请务必遵守以下规则，触犯任意一条将直接淘汰：
1. 强制 Docker 化
· web/全栈/后端：根目录必须包含 `docker-compose.yml` 文件。阅卷官执行 `docker compose up` 后，必须能一键启动并访问。
· 移动端/小程序：根目录必须包含 `docker-compose.yml` 用于启动后端服务。
· 禁止要求阅卷官在本地安装 Node/Python/Java 环境。
2. 自动化验证（CI Check）
· 我们提供了GitHub Actions 脚本。你的代码提交后，GitHub 上的 Actions状态必须显示为 [✔] Green（Success）。
3. 工程结构规范
· 拒绝“单文件代码”。请合理分层（如/frontend，/backend，/src 等）。
· 代码需要包含简要 `README.md`, 说明如何启动。
4. UI审美（针对带界面的题目）
· 拒绝浏览器默认样式。请使用组件库（Tailwindcss，AntD，Shadcn 等）保证页面整洁、对齐、美观。
